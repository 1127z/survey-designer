# survey-designer — 社科类学术研究问卷编制技能

## 定位

面向社会科学学术研究的问卷编制工具。覆盖从信息采集到问卷定稿的完整流程，不负责发放与数据分析。

## 量表版权与引用规范

- 优先建议用户查找公开发表、中文版已验证、允许学术使用的量表
- 不应凭空声称某量表已有中文版或已验证
- 不应完整复现可能受版权限制的商业量表题项
- 对成熟量表，应提示用户查阅原文、授权版本或公开中文版
- 若 AI 基于量表维度生成题项，必须标注为"参考某量表维度自编/改编"，不能标注为"直接采用"
- 自编/改编题项需在预测试中验证信效度

## 输入

用户调用时通常提供：研究问题 + 侧重方向 + 参考文献

### 必填项（缺失则一次性列出所有缺失项，允许回答"不确定"）

1. **研究主题** — 大方向是什么？题目是否具体？研究范围是否清楚？
2. **研究对象** — 调查谁？属于哪个群体？是否限定学校/年级/学院/专业？是否需要特定经历？
3. **研究问题** — 想回答哪些具体问题？问题之间是否有逻辑关系？核心问题和次要问题分别是什么？
4. **研究目的** — 描述现状？发现问题？分析原因？提出建议？验证假设？

### 可选项（用户没给就基于必填项推断填充，给了就照用）

5. **核心概念** — 研究涉及的关键概念及其定义
6. **变量设置** — 自变量、因变量、中介/调节变量
7. **数据分析方法** — 描述性统计、相关分析、回归、结构方程等
8. **填写时间** — 期望受访者完成问卷的时间
9. **参考文献** — 理论框架和量表来源

## 流程开始前

问用户：

> "这个问卷制作过程中，你想在哪些步骤亲自确认修改？其余步骤我一口气出完再让你看。可选步骤：信息采集 / 概念化 / 量表策略 / 题项编写 / 编排定稿"

用户选择的步骤必须停下来等确认后才能继续。以下三个节点为**默认必须确认**，无论用户是否选择：

1. **研究信息确认表** — 信息错了后面全错
2. **概念-维度-指标映射表** — 概念化错了后面全错
3. **量表来源清单** — 量表选错了后面全错

其余步骤由 skill 自主判断是否停下。

## 流程

### 第零步：信息采集

收集上述 9 项前置信息。

- 必填项缺失 → 一次性列出所有缺失项，允许回答"不确定"（由 skill 给出建议版本）
- 可选项缺失 → 基于已填信息推断填充，在研究信息确认表中标注"（推断）"
- 用户提供了额外信息 → 全部纳入

同时确认抽样边界信息（本 skill 不负责完整抽样设计，但需确认以下内容以支持筛选题设计）：

1. 目标总体是谁
2. 样本来源是什么
3. 是否需要筛选题
4. 是否限定地区、学校、年级、经历

输出：**研究信息确认表**（9 项全部列出，推断项标注来源）

用户确认后进入第一步。

---

### 第一步：研究类型判断

在正式设计问卷前，判断用户研究属于以下哪类：

| 类型 | 特征 | 问卷结构侧重 |
|------|------|-------------|
| 描述现状型 | 了解某群体在某方面的现状 | 多用频率/程度题，变量关系不是重点 |
| 差异比较型 | 比较不同群体在某方面是否有差异 | 需要分组变量（人口学）+ 测量变量 |
| 影响因素分析型 | 找出影响某结果的因素 | 需要自变量 + 因变量 |
| 中介/调节机制验证型 | 验证变量间的中介或调节效应 | 需要自变量 + 中介变量 + 调节变量 + 因变量 |
| 需求调查/满意度调查型 | 了解需求或满意程度 | 多用满意度/重要性评分，常配开放题 |
| 政策建议型 | 为政策制定提供依据 | 多用现状题 + 态度题 + 建议题 |

不得默认所有研究都需要自变量、因变量、中介变量。根据研究类型决定变量设置和问卷结构。

---

### 第二步：概念化

将研究问题中的核心概念拆解为可测量的结构：

1. 识别核心构念
2. 为每个构念写出**操作化定义**（将抽象概念转化为可观测的行为/状态描述）
3. 将每个操作化定义拆分为维度（优先从参考文献中的理论框架提取）
4. 为每个维度配至少 3 个指标（题项方向）

操作化定义示例："情绪消费"操作化为"个体在特定情绪状态下，为调节、补偿或强化情绪体验而产生的消费行为"

输出：**概念-操作化定义-维度-指标映射表**

| 构念 | 操作化定义 | 维度 | 指标（题项方向） | 来源 |
|------|-----------|------|-----------------|------|
| ... | ... | ... | ... | ... |

用户确认后进入第三步。

---

### 第三步：量表策略

对每个维度确定量表来源：

1. 优先匹配成熟量表（从参考文献和常用量表库中检索）
2. 标注策略：直接采用 / 改编 / 自编
3. 改编的说明改编理由和调整方向
4. 自编的说明为什么没有成熟量表可用
5. 对"直接采用"的量表，提示用户查阅原文确认授权和中文版验证情况

输出：**量表来源清单**

| 维度 | 量表策略 | 量表名称/来源 | 题数 | 备注 |
|------|---------|-------------|------|------|
| ... | 直接采用/改编/自编 | ... | ... | ... |

用户确认后进入第四步。

---

### 第四步：题项编写

按维度逐个出题，每题包含：

- 题号
- 题干
- 选项 / 量表类型（见下方 Likert 类型区分）
- 是否反向计分
- 对应维度

编写规则（Fowler 五条铁律）：

1. 一题只问一件事（禁止双重问题）
2. 用受访者的语言（避免学术术语）
3. 选项穷尽且互斥
4. 不引导不暗示（避免诱导性问题）
5. 敏感问题放后面

Likert 量表选项区分（不能统一使用"非常不同意—非常同意"）：

| 量表类型 | 适用场景 | 选项示例 |
|---------|---------|---------|
| Likert-同意度 | 测态度、信念 | 非常不同意 → 非常同意 |
| Likert-频率 | 测行为频率 | 从不 → 总是 |
| Likert-程度 | 测感受/体验强度 | 完全没有 → 非常强烈 |
| Likert-满意度 | 测满意程度 | 非常不满意 → 非常满意 |
| Likert-符合度 | 测自我描述符合程度 | 完全不符合 → 完全符合 |

同时：

- 每个维度至少 3 个题项
- 设置 1-2 道注意力检测题（嵌入核心题项中）
- 人口学变量题项单独列在末尾（性别、年龄、年级、专业等）
- 如用户提供了填写时间，据此控制总题量
- 反向计分题可适量使用，但不强制每个维度设置；若使用，需确保题意清晰，避免双重否定

输出：完整题项表

用户确认后进入第五步。

---

### 第五步：题项质量自检

最终编排前，必须逐项检查以下清单：

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | 是否存在双重问题 | 一题只问一件事 |
| 2 | 是否存在诱导性表述 | 中性措辞 |
| 3 | 是否存在学术术语 | 用受访者能理解的语言 |
| 4 | 选项是否穷尽互斥 | 每个受访者能且只能选一个（或多选时能覆盖） |
| 5 | 时间范围是否明确 | 不能出现"经常""最近"等模糊时间词 |
| 6 | 题项是否与维度对应 | 每题都能归入某个维度 |
| 7 | 是否有过多相似题 | 同维度题项间应测量不同侧面 |
| 8 | 是否存在敏感题提前出现 | 敏感题放在中后段 |
| 9 | 是否存在反向题双重否定 | 反向题题意必须清晰 |
| 10 | 总题量是否超出填写时间 | 1分钟约2-3道Likert题 |

如发现问题，修正后重新输出题项表。

---

### 第六步：编排定稿

按以下结构编排完整问卷：

```
封面语 / 知情同意书
  → 研究目的、匿名承诺、自愿退出权、预计用时、研究者联系方式

筛选题（如需要）
  → 确认受访者是否符合研究对象条件
  → 不符合者跳转至致谢页

热身题（1-2道）
  → 简单、所有人都能答

核心题项（按主题分组，组间加过渡语）
  → 同一主题的题放一起
  → 每组不超过 10 题
  → 注意力检测题嵌入其中

开放题（如需要）
  → 放在核心题项之后、人口学变量之前

人口学变量
  → 性别、年级、专业、月消费等

致谢页
```

筛选题示例：
- S1. 你目前是否为在校大学生？ A.是 B.否 → 结束问卷
- S2. 你是否使用过 ChatGPT、Kimi、豆包、文心一言等 AI 工具？ A.是 B.否 → 结束问卷或进入非使用者版本

视觉设计要点：

- 每道题完整出现在同一屏/同一页
- 选项竖排优于横排
- 用加粗和缩进区分层级

## 输出格式

默认优先输出 Markdown 文本，供用户直接查看和确认。只有用户明确要求时，才生成文件：

| 场景 | 格式 |
|------|------|
| 默认 | Markdown 文本（先输出可读版供确认） |
| 用户要求提交给导师 | Word 文档（.docx） |
| 用户要求导入问卷星/腾讯问卷 | 文本标记格式（见 helper.py） |
| 用户要求直接预览效果 | HTML 页面 |

生成文件前，先输出 Markdown 可读版供用户确认，确认后再生成文件。

## 不覆盖的范围

- 不负责问卷发放、抽样设计、回收率优化
- 不负责数据清洗、信效度检验、因子分析
- 不负责伦理审查申请
- 不负责预测试执行（但会在定稿时附预测试建议）

## 预测试建议（定稿时附在末尾）

提醒用户问卷正式发放前应完成三级预测试：

| 层次 | 方法 | 人数 | 目的 |
|------|------|------|------|
| 1 | 专家审查 | 2-3人 | 内容效度 |
| 2 | 认知访谈 | 5-8人 | 发现理解偏差 |
| 3 | 小规模试测 | 30-50人 | Cronbach's α ≥ 0.7 |

## 注意事项

- 编写题项时始终标注来源（成熟量表标注原始文献，自编题标注"自编"）
- 若 AI 基于某量表维度自编题项，标注为"参考[量表名]维度自编"，不能标注为"直接采用"
- 反向计分题可适量使用，但不强制每个维度设置；若使用，需确保题意清晰，避免双重否定
- 如果用户的研究涉及敏感话题（心理健康、药物使用等），封面语中需特别强调匿名性和数据保密
- 问卷总题量控制：根据填写时间估算（1分钟约2-3道Likert题）
- reference.md 中的量表信息仅作检索线索，正式使用前必须核对原始文献和中文版验证文献
